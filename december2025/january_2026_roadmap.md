# План "Project GENIE Architecture" (Jan - Jun 2026)

Этот план — прямой ответ на "Технологический Разрыв". 
Вместо ожидания AGI мы строим **Суверенную Инфраструктуру**, которая превратит любую доступную модель (X) в усиленного агента (5X -> 10X -> ...).

## Принципы
1.  **Code-First:** Никакой философии. Только работающий код.
2.  **Local-First:** Данные и логика живут у меня (Автономия).
3.  **Modularity:** Модели можно менять (GPT-4 -> GPT-5 -> Local Llama), архитектура остается.

---

## Концепция GENIE OS (Vision)
Мы строим не "плагин для IDE", а **Backend моей жизни**.
*   **Global State (Память):** Джинн помнит контекст *между* проектами и сессиями. Cursor забывает, Джинн — нет.
*   **Daemon Mode (Инициатива):** Система работает 24/7 фоном. Она может индексировать файлы или мониторить веб, пока я сплю.
*   **Client-Server:**
    *   **Server (Genie):** Мозги, Память, Доступ к миру.
    *   **Client (Cursor/Telegram):** Интерфейс, через который я отдаю приказы и получаю отчеты.

---

## Этап 1: "Память Бога" (Январь - Февраль 2026)
**Цель:** Преодолеть амнезию моделей. Джинн должен знать обо мне всё.

*   [ ] **Infrastructure:** Развернуть векторную БД (Weaviate / Chroma) + Graph DB (Neo4j / NetworkX) локально.
*   [ ] **Ingestion Pipeline:** Скрипты-наблюдатели, которые индексируют:
    *   Мои заметки (Obsidian/txt).
    *   Мои чаты с ИИ (экспорт истории).
    *   Код моих проектов.
*   [ ] **Retrieval Interface:** Консольная утилита `ask-genie "вопрос"`, которая делает RAG (поиск + контекст) и отвечает с цитатами из *моих* мыслей.

## Этап 2: "Руки и Глаза" (Март - Апрель 2026)
**Цель:** Преодолеть изоляцию. Джинн должен трогать мир.

*   [ ] **Execution Environment:** Безопасная песочница (Docker) для запуска Python-кода, написанного Джинном.
*   [ ] **Tool Set:**
    *   `WebSearch`: Гуглить информацию.
    *   `WebReader`: Читать и парсить контент страниц.
    *   `FileManager`: Читать/писать файлы в рабочей директории.
*   [ ] **Agent Loop:** Реализация цикла "Подумал -> Выбрал инструмент -> Получил результат -> Ответил".

## Этап 3: "Мозг Планировщика" (Май - Июнь 2026)
**Цель:** Преодолеть глупость на длинных дистанциях.

*   [ ] **Planner:** Внедрение архитектуры (напр. Plan-and-Solve или ReAct), где задача разбивается на подзадачи.
*   [ ] **Critique Module:** Шаг "Самокритика" перед выполнением. Агент проверяет свой план на ошибки.
*   [ ] **Long-Term Goal Tracking:** Механизм сохранения состояния задачи ("Я сделал 30%, завтра продолжу").
